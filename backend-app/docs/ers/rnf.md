# Requerimientos no funcionales (RNF)

- **RNF-01** Seguridad
El sistema debe implementar mecanismos de autenticación mediante módulos que utilicen encriptación asimétrica (clave pública y privada) para proteger las credenciales de acceso. Además, debe permitir actualizar los métodos de seguridad en caso de detectar vulnerabilidades o fallas.

- **RNF-02** Velocidad
El sistema debe garantizar tiempos de respuesta óptimos, de manera que las operaciones más comunes (registro, consulta y carga de información) no superen los 3 segundos bajo condiciones normales de red. Asimismo, debe optimizar el uso de recursos del servidor para asegurar un rendimiento eficiente incluso con múltiples usuarios concurrentes.

- **RNF-03** Disponibilidad
El sistema debe asegurar una disponibilidad mínima del 99% durante las horas operativas, permitiendo su acceso continuo tanto desde equipos internos como externos a la red institucional.

- **RNF-04** Usabilidad
La interfaz del sistema debe ser intuitiva y fácil de usar, permitiendo que los usuarios sin conocimientos técnicos avanzados puedan realizar sus tareas con una mínima curva de aprendizaje. Se priorizará un diseño claro, accesible y compatible con distintos dispositivos y resoluciones.

- **RNF-05** Mantenibilidad
El sistema debe estar diseñado con una arquitectura modular que facilite la corrección de errores, actualización de componentes y ampliación de funcionalidades sin afectar la estabilidad del sistema general.

- **RNF-06** Compatibilidad
El sistema debe ser compatible con los principales navegadores web modernos (Chrome, Firefox, Edge) y adaptarse correctamente tanto a entornos de escritorio como móviles.

 
- **RNF-07** Experiencia de Usuario
El sistema debe respetar la paleta de colores de la institución para que coincida con el resto de aplicaciones propias de la institución, asegurando consistencia visual y mejor usabilidad. Esto incluye:

Rojo institucional: #D32F2F

Blanco institucional: #FFFFFF

Tipografía: Usar fuente oficial de la institución en todos los textos (por ejemplo, Arial o la fuente corporativa).

Interfaz: Los botones, enlaces y alertas deben respetar la jerarquía visual establecida por la paleta de colores.

Consistencia: Todos los módulos y pantallas deben mantener la misma línea visual para evitar confusión del usuario.
