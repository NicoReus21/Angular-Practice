# Requerimientos

Este directorio centraliza las Tarjetas de Requerimiento para documentar y seguir el progreso de funcionalidades, cambios o tareas.

## Estructura

- Cada requerimiento es un archivo Markdown independiente dentro de `docs/requirements/`.
- Usa el template `TEMPLATE.md` para mantener el estándar.
- Opcionalmente crea subcarpetas por área o módulo si lo necesitas.

## Nomenclatura sugerida

- `REQ-0001-nombre-corto.md` (usa ceros a la izquierda y un slug descriptivo)
- Ejemplos: `REQ-0007-reporte-flash.md`, `REQ-0123-mantenimiento-autos.md`

## Flujo recomendado

1. Copia `TEMPLATE.md` a un nuevo archivo (p. ej. `REQ-0001-nombre.md`).
2. Completa los campos obligatorios y la sección de Descripción.
3. Envía PR asociando el archivo; actualiza la sección de Estado según avance.
4. Mantén `Última actualización` al día.

## Template

El template oficial está en `docs/requirements/TEMPLATE.md`.
